## 1. 数据模型

**文件**: `python/models/schemas.py`

这是整个系统的"语言"——定义了所有 Agent 之间传递的数据结构。

### 为什么用 Pydantic？

```python
from pydantic import BaseModel, Field

class UserPreferences(BaseModel):
    budget: float = Field(..., gt=0, description="总预算（人民币元）")
    travel_style: TravelStyle = Field(default=TravelStyle.COMFORT)
```

- `Field(..., gt=0)`: 自动校验 budget 必须大于 0，无效输入直接报错
- 类型注解 + Pydantic = **运行时类型检查**，比普通 dict 安全得多
- `model_dump()` 自动序列化为 JSON，方便 API 传输

### 核心状态对象

```python
class TravelPlanState(BaseModel):
    state: PlanningState          # 当前处于哪个阶段
    preferences: UserPreferences  # 由 PreferenceAgent 填充
    flight_result: FlightSearchResult    # 由 FlightAgent 填充
    hotel_result: HotelSearchResult      # 由 HotelAgent 填充
    activity_result: ActivitySearchResult # 由 ActivityAgent 填充
    budget_breakdown: BudgetBreakdown    # 由 BudgetAgent 填充
    adjustment_round: int = 0            # 当前调整轮次
```

**设计要点**: 每个 Agent 只写自己负责的字段，读取其他 Agent 写入的字段。
这样并行执行时不会有写冲突（Flight 写 flight_result，Hotel 写 hotel_result）。

## 2. Agent 基类

**文件**: `python/agents/base_agent.py`

### 模板方法模式

```python
class BaseAgent(ABC):
    async def run(self, state):
        logger.info(f"[{self.name}] 开始执行...")
        try:
            state = await self.execute(state)
        except Exception as exc:
            state.error_messages.append(f"{self.name}: {str(exc)}")
        return state

    @abstractmethod
    async def execute(self, state):
        ...
```

- `run()` 是模板方法，定义了"日志→执行→错误处理"的固定流程
- `execute()` 是抽象方法，子类必须实现自己的业务逻辑
- 好处：新增 Agent 只需实现 `execute()`，日志和错误处理自动拥有

### Mock / 真实 LLM 双模式

```python
async def call_llm(self, prompt, system_prompt=""):
    if self._llm_provider == "mock":
        return self._mock_llm(prompt)    # 零成本演示
    return await self._real_llm(prompt)  # 真实 API 调用
```

**面试讲法**: "系统默认用 Mock 模式运行，不需要任何 API Key 就能完整演示。
切换到真实模式只需要设置环境变量 `LLM_PROVIDER=minimax`。"

## 3. 偏好收集 Agent

**文件**: `python/agents/preference_agent.py`

```python
class PreferenceAgent(BaseAgent):
    async def execute(self, state):
        pref = state.preferences
        if not pref.interests:
            pref.interests = self._default_interests(pref.travel_style.value)
        state.state = PlanningState.RECOMMENDING_DESTINATIONS
        return state
```

**核心逻辑**: 如果用户没有提供兴趣标签，根据旅行风格自动补充。

| 风格 | 自动补充的兴趣 |
|------|--------------|
| budget | 免费景点、街头美食、步行游览 |
| luxury | 米其林餐厅、私人导游、SPA |
| cultural | 博物馆、历史遗迹、传统手工艺 |

**面试考点**: 为什么要单独一个 Agent 做偏好收集？
→ 在真实场景中，偏好收集可能需要多轮对话（"你想去海边还是山里？"），逻辑很复杂。

## 4. 目的地推荐 Agent

**文件**: `python/agents/destination_agent.py`

### 多维度评分算法

```python
def _score_destination(dest, budget, style, start_date):
    score = 0.0
    # 预算匹配度 (30分)
    if budget >= estimated_cost:
        score += 30
    # 安全评分 (30分)
    score += dest.safety_score * 3
    # 季节适宜度 (20分)
    if current_season in dest.best_season:
        score += 20
    # 风格匹配度 (15分)
    if style matches dest.cost_level:
        score += 15
    # 免签加分 (10分)
    if not dest.visa_required:
        score += 10
```

**面试讲法**: "评分函数考虑 5 个维度，权重分别是 30/30/20/15/10，
总分 105 分。这个权重是通过分析用户反馈数据调优得到的（Mock 数据中是人工设定的）。"

## 5. 航班搜索 Agent

**文件**: `python/agents/flight_agent.py`

### 航班推荐评分

```python
def score(f: Flight):
    price_score = 1 - (f.price / max_price)    # 价格越低分越高
    dur_score = 1 - (f.duration / max_dur)      # 时间越短分越高
    stop_score = 1 - (f.stops / 3)              # 中转越少分越高
    budget_bonus = 10 if f.price <= budget_share else 0
    return price_score * 50 + dur_score * 30 + stop_score * 20 + budget_bonus
```

**权重设计**: 价格 50% > 时长 30% > 中转 20%，符合大多数用户"价格敏感"的特征。

## 8. 预算校验 Agent

**文件**: `python/agents/budget_agent.py`

### 渐进式降级

```python
if round_num == 1:  # 第1轮: 砍活动
    for act in activities:
        act.price *= (1 - cut_ratio)

elif round_num == 2:  # 第2轮: 降酒店
    hotel.price_per_night *= (1 - cut_ratio)

elif round_num >= 3:  # 第3轮: 换航班
    flight.price *= (1 - cut_ratio)
```

**cut_ratio 的计算**:
```python
cut_ratio = min(0.4, over_amount / category_cost)
```
- 超预算金额 / 该类别总费用 = 需要削减的比例
- 但最多不超过 40%，避免过度削减影响体验

## 9. 并行执行器

**文件**: `python/orchestrator/parallel.py`

```python
class ParallelExecutor:
    async def run(self, state):
        tasks = [
            asyncio.wait_for(agent.run(state), timeout=self.timeout)
            for agent in self.agents
        ]
        results = await asyncio.gather(*tasks, return_exceptions=True)
```

**三个关键点**:
1. `asyncio.gather`: 同时启动所有任务
2. `return_exceptions=True`: 某个 Agent 失败不影响其他
3. `asyncio.wait_for + timeout`: 防止单个 Agent 卡住整个系统

## 10. 预算循环

**文件**: `python/orchestrator/budget_loop.py`

```python
for attempt in range(self.max_retries + 1):
    # 并行搜索
    state = await self.parallel_executor.run(state)
    # 预算校验
    state = await self.budget_agent.run(state)
    # 检查是否通过
    if state.state == PlanningState.COMPLETED:
        return state  # 预算通过，退出循环
    # 否则进入下一轮调整
```

**循环体做了什么？**
1. 并行执行 Flight + Hotel + Activity
2. Budget Agent 汇总费用、比较预算
3. 超预算 → 应用降级策略 → adjustment_round += 1 → 下一轮
4. 预算通过 → 设置 state = COMPLETED → 退出

## 11. Pipeline 编排器

**文件**: `python/orchestrator/pipeline.py`

```python
class TravelPlanningPipeline:
    async def run(self, preferences):
        state = TravelPlanState(preferences=preferences)

        # 阶段1: 偏好收集（串行）
        state = await self.preference_agent.run(state)

        # 阶段2: 目的地推荐（串行）
        state = await self.destination_agent.run(state)

        # 阶段3: 并行搜索 + 预算循环
        state = await self.budget_loop.run(state)

        return state
```

**三行代码就是整个系统的骨架**。这就是好架构的力量。
